Video -> Character Card (Sticky Token)

Flow (service API):
1) Create video
   POST /v1/videos
2) Get generation_id
   GET /v1/videos/{id}
   -> metadata.generation_id = gen_xxx
3) Create character card from generation
   POST /v1/characters/from-generation
   Body: {
     "generation_id": "gen_xxx",
     "timestamps": [0, 4],
     "username": "my_character",
     "display_name": "My Character"
   }

Notes:
- Sticky token: the character card uses the same account that created the generation_id.
- Username availability check uses: /project_y/profile/username/check (same logic as direct character creation).
- timestamps supports "0,4" or [0,4].

Image -> Character (one call):
- POST /v1/characters with input_image or input_reference.
- Service runs image-to-video, then uses generation_id to create the character card (sticky token).

Upstream reference:
- POST https://sora.chatgpt.com/backend/characters/from-generation
- GET  https://sora.chatgpt.com/backend/project_y/cameos/in_progress/{cameo_id}
- POST https://sora.chatgpt.com/backend/project_y/profile/username/check
